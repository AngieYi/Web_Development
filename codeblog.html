  • I've learnt from the process of solving the following problems.

    • 1. the root path ('/')is routed to the index page, and the appropriate data is provided to fill out all details of the index page template (user selection list).

    Solutions: Sever has a .json file which has the several users information in it, including name and things to do. Client enters root path('/') in the browser, means sending a request to the server, server gets this request, and response.render .json data to index-page.handlerbars. In index-page.handlerbars, there is user-select.handlebars,{{> user-select }}. In the user-select.handlebars, should set each .json 's key = each select option value, .json name = select option name.

    2. the dynamic route '/notes/' routes to the corresponding user's notes page, and the appropriate data is provided to fill out all details of the notes page template (all of the user's pre‐defined notes﴿

    Solutions: when the client select one user, index.js listened a change event, will handleUserSelection(event), since each option's value is .json's key, so event.target.value will be the key, after window.location.href = '/notes/' + event.target.value, the URL in the browser will be changed, when url changes, the sever will get the request.

    According to different key (request.parameter.var_userkey), sever will response render this specific user's things to do (.json 's notes) to notes-page.handlerbars. In notes-page.handlerbars, there is a todo.handlebars {{> todo }}. Each note content corresponds to one todo note, so needs to loops all the possible notes in notes-page.handlerbars. In todo.handlebars, use some css style to decorate it, and put the notes content in right places.

    3. The function generateTodoHTML() is correctly implemented, so that new todo notes can be added on the client.

    Solution: In notes-page.handlerbar, there is an add-note-button-modal.handlerbars {{> add-note-button-modal}}. When client click the add-note-button, will load add-note-modal, all the events are handled by index.js, after click Add ToDo, then function insertNewTodo() will be called, in this function, there is generateTodoHTML(), written in generate-todo.js, it uses the template on the client side to generate HTML for a new note to be added into the document.

    The template was built when service started.

      "build": "handlebars views/partials/todo.handlebars -f public/todo-template.js",
      "prestart": "npm run build",

    More specifically, in generate-todo.js

      var todoTemplate = Handlebars.templates.todo; // makes that template available on the client side.

    todoTemplate is actually a function. Needs to pass the correct data into that function to fill out the template. The data format should keep the same as the format used in todo.handlebars. The todoTemplate(content) function will return an HTML string, and the HTML string is handled by insertNewTodo() in index.js.
